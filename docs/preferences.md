# Preferences

This page documents every field in `preferences.json`, the workspace configuration file that controls commit style, branch naming, worktree patterns, and other conventions.

## Overview

`preferences.json` lives at the root of every agent workspace. It is the single source of truth for workspace conventions. Both scripts (via `jq`) and AI agents (via `TOOLS.md` references) read this file.

To change a preference, edit `preferences.json` directly. Changes take effect immediately — there is no build step or restart required.

## Field Reference

### Top-level

| Field | Type | Description |
|-------|------|-------------|
| `agent_name` | string | The agent's name (lowercase). Set during init. |
| `agent_display_name` | string | The agent's display name (capitalized). Set during init. |

### commits

Controls how the agent writes git commit messages.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `commits.style` | string | `"conventional"` | Commit message style. Currently only `"conventional"` is supported (e.g., `feat:`, `fix:`, `chore:`). |
| `commits.include_footer` | boolean | `false` | Whether to include a footer line (e.g., "Generated by AI") in commit messages. |
| `commits.include_co_authored_by` | boolean | `false` | Whether to include `Co-authored-by` trailers in commits. |
| `commits.sign_off` | boolean | `false` | Whether to add a `Signed-off-by` line to commits. |
| `commits.skip_hooks_env` | string | `""` | Environment variable to set when committing in worktrees where pre-commit hooks may fail. Example: `"SKIP_GIT_HOOKS=1"`. When set, the agent prepends this to commit commands. |

### branches

Controls branch naming conventions.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `branches.prefix` | string | `""` | Prefix prepended to all branch names. Example: `"yourname/"` produces branches like `yourname/fix-something`. |
| `branches.format` | string | `"{prefix}{name}"` | Branch name format. Uses `{prefix}` and `{name}` as placeholders. |

### worktrees

Controls how git worktrees are named and organized.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `worktrees.naming` | string | `"{repo}--{branch}"` | Naming pattern for worktree directories. Uses `{repo}` and `{branch}` as placeholders. Example: `dashboard--yourname/fix-bug`. |
| `worktrees.base_dir` | string | `"workspaces"` | Directory within the workspace where worktrees are created. |

### subagents

Controls parallelism limits for spawned subagents.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `subagents.max_parallel_read` | number | `5` | Maximum number of subagents that can run in parallel for read-only operations (research, review, exploration). |
| `subagents.max_parallel_write` | number | `1` | Maximum number of subagents that can run in parallel for write operations (builds, tests, implementation). Keep this at 1 to avoid resource contention. |

### session

Controls session lifecycle behavior.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `session.auto_commit` | boolean | `true` | Whether `session-end.sh` should auto-commit workspace changes (tasks, memory, plans). |
| `session.daily_memory_logs` | boolean | `true` | Whether the agent should write daily memory logs to `memory/YYYY-MM-DD.md`. |

### github

GitHub-related configuration.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `github.org` | string | `""` | GitHub organization name. Used by scripts that interact with GitHub (e.g., PR creation, status sync). |

## How Scripts Read Preferences

### From bash (using jq)

```bash
BRANCH_PREFIX=$(jq -r '.branches.prefix' preferences.json)
COMMIT_STYLE=$(jq -r '.commits.style' preferences.json)
MAX_PARALLEL=$(jq -r '.subagents.max_parallel_read' preferences.json)
```

### From Node.js

```javascript
import { readFileSync } from 'node:fs'

const prefs = JSON.parse(readFileSync('preferences.json', 'utf-8'))
console.log(prefs.branches.prefix)
```

## How the Agent Reads Preferences

The agent learns about preferences through `TOOLS.md`, which says:

> Workspace conventions (commit style, branch naming, etc.) are defined in `preferences.json` — edit that file to customize.

Each section in `TOOLS.md` references the relevant preference key. For example, the "Commits" section tells the agent to follow the style defined in `preferences.json` under `commits`.

## Examples of Common Customizations

### Add a branch prefix

```json
{
  "branches": {
    "prefix": "alice/",
    "format": "{prefix}{name}"
  }
}
```

Result: branches are created as `alice/fix-something`, `alice/add-feature`.

### Skip pre-commit hooks in worktrees

```json
{
  "commits": {
    "skip_hooks_env": "SKIP_GIT_HOOKS=1"
  }
}
```

Result: the agent runs `SKIP_GIT_HOOKS=1 git commit -m "..."` when committing in worktrees.

### Reduce subagent parallelism

```json
{
  "subagents": {
    "max_parallel_read": 3,
    "max_parallel_write": 1
  }
}
```

Result: the agent spawns at most 3 parallel subagents for read-only tasks.

### Disable auto-commit on session end

```json
{
  "session": {
    "auto_commit": false
  }
}
```

Result: `session-end.sh` validates tasks and memory but does not commit changes automatically.
